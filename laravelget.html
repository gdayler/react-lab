app/Models/Mascota.php
La clase PHP que representa tu entidad. Le dice a Laravel que existe una tabla mascotas y permite hacer consultas sin SQL.
database/migrations/
Define las columnas de la tabla. Sin ejecutar esto, la tabla no existe en la BD.
routes/web.php
Cuando el usuario entra a /mascotas, aquí se consultan los datos y se envían a la vista.

.env
Aquí configuras la conexión a MySQL: nombre de BD, usuario, contraseña.
04/15Paso 1 — .env

Conectar Laravel a MySQL

Laravel 11 usa SQLite por defecto. Debes cambiarlo a MySQL. Abre el archivo .env en la raíz del proyecto y modifica estas líneas:

DB_CONNECTIONmysql← cambiar de sqlite a mysql
DB_HOST127.0.0.1← dejar igual
DB_PORT3306← dejar igual
DB_DATABASEveterinaria← nombre de tu BD sin espacios
DB_USERNAMEroot← tu usuario MySQL
DB_PASSWORDtu_password← tu contraseña MySQL
📌 El docente usa veterinaria. Tú pon el nombre de tu proyecto en minúsculas sin espacios: mercado, cursosapp, clinica.
⚠ El .env nunca se sube a GitHub. Tiene contraseñas del proyecto.
05/15Paso 2 — Migrate inicial

Crear la base de datos

Con el .env configurado, ejecuta en la terminal:

php artisan migrate

Laravel detecta que la BD no existe y pregunta si crearla. Escribe yes y Enter:

WARN  The database 'veterinaria' does not exist on the 'mysql' connection.
Would you like to create it? (yes/no) [yes]
❯ yes
INFO  Running migrations.
  create_users_table ..... DONE
  create_cache_table ..... DONE
  create_jobs_table ...... DONE
💡 Laravel crea la BD y 3 tablas del sistema: users, cache, jobs. No las toques, son de Laravel.
✓ Abre DBeaver y verifica que la BD fue creada con esas 3 tablas.
06/15Paso 3 — Modelo

Crear modelo y migración

Un solo comando crea los dos archivos. El -m significa «también crea la migración».

php artisan make:model Mascota -m
🐾 El docente usa Mascota. Tú usa el nombre de tu entidad en singular con mayúscula: Producto, Curso, Cliente.
📄

app/Models/Mascota.php

La clase PHP del modelo. Debes editarla para agregar $fillable con tus campos. Sin eso no puedes insertar datos.

📋

database/migrations/
xxxx_create_mascotas_table.php

Define las columnas de la tabla. Debes editarlo para agregar tus campos antes de ejecutar migrate.

⚠ Verifica en VSCode que los 2 archivos existen antes de continuar.
07/15Paso 4 — Modelo completo

Editar el modelo — agregar $fillable

Abre app/Models/Mascota.php. El archivo completo debe quedar así:

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Mascota extends Model
{
    protected $fillable = ['nombre', 'especie', 'edad', 'peso'];
}
¿Qué es $fillable?
Es la lista de campos que Laravel permite insertar o editar. Sin esta línea, si intentas insertar datos Laravel bloquea la operación y lanza un error de seguridad llamado «mass assignment».
📌 Los nombres dentro del array deben ser exactamente iguales a los que pondrás en la migración.

Ejemplo para Curso:
protected $fillable = ['titulo', 'precio', 'duracion_horas', 'nivel'];
08/15Paso 5 — Migración

Editar la migración y crear la tabla

Abre el archivo de migración. Dentro de Schema::create agrega tus campos:

Schema::create('mascotas', function (Blueprint $table) {
    $table->id();
    $table->string('nombre');
    $table->string('especie');
    $table->integer('edad');
    $table->decimal('peso', 5, 2);
    $table->timestamps();
});
⚠ No borres $table->id() ni $table->timestamps().

Luego ejecuta:

php artisan migrate
✓ Verifica en DBeaver que tu tabla aparece con los campos correctos.

Tipos de columnas:

string('campo')Texto corto. Nombres, títulos.
text('campo')Texto largo. Descripciones.
integer('campo')Número entero. Edad, stock.
decimal('campo',8,2)Decimal. Precios, peso.
boolean('campo')Verdadero/falso. Activo.
date('campo')Fecha. Nacimiento.
09/15Paso 6 — Insertar datos

Agregar registros en DBeaver

Antes de probar la vista necesitas datos en la tabla. Agrégalos desde DBeaver:

1
Abre DBeaver y conecta a tu BD

Selecciona la base de datos configurada en el .env

2
Expande la BD y busca tu tabla

Haz doble clic sobre mascotas para abrirla

3
Clic en el botón + de la barra inferior

Aparece una fila nueva vacía. Llena los campos: nombre, especie, edad, peso.

4
Guarda con Ctrl+S

DBeaver pedirá confirmación. Acepta. Repite para tener mínimo 3 registros.

✓ Verifica que los 3 registros aparecen en la tabla de DBeaver antes de continuar.
10/15Paso 7 — Ruta

Consultar datos y enviarlos a la vista

Abre routes/web.php. El archivo completo debe quedar exactamente así:

<?php

use Illuminate\Support\Facades\Route;
use App\Models\Mascota;  // ← SIEMPRE al inicio, después de <?php

Route::get('/', function () {
    return view('welcome');
});

Route::get('/mascotas', function () {
    $mascotas = Mascota::all();  // trae todos los registros de la tabla
    return view('mascotas.index', ['mascotas' => $mascotas]);
    // ↑ envía los datos a la vista con el nombre $mascotas
});
Mascota::all()
Trae todos los registros de la tabla mascotas. Laravel ejecuta automáticamente SELECT * FROM mascotas. No necesitas escribir SQL.
['mascotas' => $mascotas]
Envía los datos a la vista. Dentro de la vista tendrás disponible la variable $mascotas con todos los registros.
⚠ El use App\Models\Mascota DEBE estar al inicio del archivo después de <?php. Si lo pones en otro lugar Laravel dará error de sintaxis.
11/15Paso 8 — Vista

Crear la vista con tabla HTML

Crea la carpeta resources/views/mascotas/ y dentro el archivo index.blade.php.

⚠ Laravel 11 sin Breeze NO tiene layouts. No uses @extends ni @section. Usa HTML completo directo.
<!DOCTYPE html>
<html>
<head>
    <title>Lista de Mascotas</title>
</head>
<body>
    <h1>Lista de Mascotas</h1>

    <table border="1" cellpadding="8" cellspacing="0">
        <thead>
            <tr>
                <th>ID</th>
                <th>Nombre</th>
                <th>Especie</th>
                <th>Edad</th>
                <th>Peso</th>
            </tr>
        </thead>
        <tbody>
            @foreach($mascotas as $mascota)
            <tr>
                <td>{{ $mascota->id }}</td>
                <td>{{ $mascota->nombre }}</td>
                <td>{{ $mascota->especie }}</td>
                <td>{{ $mascota->edad }}</td>
                <td>{{ $mascota->peso }}</td>
            </tr>
            @endforeach
        </tbody>
    </table>
</body>
</html>
12/15Blade explicado

¿Qué hace cada parte de la vista?

@foreach($mascotas as $mascota)
Repite el bloque HTML por cada registro de la BD. Si hay 3 mascotas, crea 3 filas. $mascota representa cada registro uno por uno.
{{ $mascota->nombre }}
Muestra el valor del campo nombre del registro actual. Cambia nombre por cualquier campo de tu tabla: precio, edad, etc.
@endforeach
Cierra el bloque. Todo lo que está entre @foreach y @endforeach se repite por cada registro.
📌 Cambia los nombres de columnas por los campos de tu propia entidad. Ejemplo: {{ $curso->titulo }}, {{ $curso->precio }}

13/15Errores comunes

Errores y soluciones

!
Table doesn't exist

No ejecutaste migrate después de editar la migración. Ejecuta php artisan migrate.

!
Table already exists

La tabla ya existía. Ejecuta php artisan migrate:fresh para borrar y recrear todo.

!
View [layouts.app] not found

Estás usando @extends pero no existe el layout. Usa HTML completo directo sin @extends ni @section.

!
Undefined variable $mascotas

No enviaste los datos a la vista. Verifica que la ruta tiene ['mascotas' => $mascotas] como segundo parámetro del view().

!
La tabla aparece vacía en el navegador

No hay registros. Inserta datos desde DBeaver siguiendo el Paso 6.

!
Error de sintaxis con use App\Models\Mascota

El use está en medio del archivo. Debe ir al inicio, justo después de <?php.
