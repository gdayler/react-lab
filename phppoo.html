Orientada a Objetos

Antes de tocar Laravel, necesitamos entender POO. Es el paradigma en que se basa todo — clases, objetos, herencia, encapsulamiento.

⏱ Duración: 25 minutos

Bloque 1 · Concepto

¿Qué es la Programación Orientada a Objetos?

🏗️

Clase

Es el molde o plano. Define qué propiedades y comportamientos tendrá un objeto. Ejemplo: la clase Curso.

📦

Objeto

Es una instancia de una clase. Cada curso concreto (PHP, React, Laravel) es un objeto de la clase Curso.

🔧

Método

Son las funciones que puede hacer el objeto. Ejemplo: inscribir(), calcularPrecio().

🔒

Encapsulamiento

Proteger los datos internos. Se usa private para que solo la propia clase pueda modificarlos directamente.

🧬

Herencia

Una clase puede heredar propiedades y métodos de otra. CursoGratuito puede heredar de Curso.

🎭

Polimorfismo

Un método puede comportarse diferente según la clase que lo implementa. calcularPrecio() en gratuito devuelve 0.

💡 La analogía

Pensá en un formulario de curso: el formulario en blanco es la clase. Cada curso que llenás con ese formulario es un objeto.

Bloque 1 · Código

Tu primera clase en PHP

Copiá esto en un archivo nuevo. Cada comentario explica exactamente qué hace cada parte:

Curso.php — tu primera clase
<?php
// ┌─────────────────────────────────────────────────────────────┐
// │  CLASE: es el "molde". Define estructura + comportamiento.  │
// │  No guarda datos reales — eso lo hace el OBJETO.            │
// └─────────────────────────────────────────────────────────────┘
class Curso
{
    // ── PROPIEDADES ────────────────────────────────────────────
    // Son las "variables" que pertenecen a cada objeto.
    // "private" significa que solo esta clase puede leerlas/modificarlas
    // desde afuera. Esto se llama ENCAPSULAMIENTO.
    private string $titulo;         // string = texto
    private string $descripcion;
    private float  $precio;          // float  = número con decimales
    private int    $duracion_horas;  // int    = número entero

    // ── CONSTRUCTOR ────────────────────────────────────────────
    // PHP llama a __construct() automáticamente cuando hacés:
    //   $curso = new Curso(...)
    // Los parámetros que recibe se asignan a las propiedades.
    // La sintaxis "named arguments" (titulo: 'PHP') es PHP 8+.
    public function __construct(
        string $titulo,
        string $descripcion,
        float  $precio,
        int    $duracion_horas
    ) {
        // $this hace referencia al objeto actual que se está creando.
        // $this->titulo = "el título DE ESTE objeto específico"
        $this->titulo          = $titulo;
        $this->descripcion     = $descripcion;
        $this->precio          = $precio;
        $this->duracion_horas  = $duracion_horas;
    }

    // ── GETTERS ────────────────────────────────────────────────
    // Como las propiedades son "private", necesitamos métodos
    // "getter" para leerlas desde afuera de la clase.
    // Convención de nombre: get + NombrePropiedad()
    public function getTitulo(): string
    {
        return $this->titulo;
        // ": string" después del paréntesis = tipo de retorno
        // PHP verificará que siempre devuelvas un string
    }

    public function getPrecio(): float
    {
        return $this->precio;
    }

    // ── MÉTODO CON LÓGICA ──────────────────────────────────────
    // Los métodos pueden recibir parámetros y hacer cálculos.
    // Acá calculamos el precio final con un descuento porcentual.
    // Ejemplo: precioConDescuento(20) con precio 150 → devuelve 120
    public function precioConDescuento(int $porcentaje): float
    {
        // Fórmula: precio * (1 - porcentaje/100)
        // Con 20%: 150 * (1 - 0.20) = 150 * 0.80 = 120
        return $this->precio * (1 - $porcentaje / 100);
    }

    // ── MÉTODO DE PRESENTACIÓN ─────────────────────────────────
    // Devuelve un texto legible con los datos del curso.
    // Las {} dentro del string permiten interpolar variables.
    public function resumen(): string
    {
        return "{$this->titulo} — Bs. {$this->precio} — {$this->duracion_horas}h";
    }
}

// ══ USAR LA CLASE ══════════════════════════════════════════════
// "new Curso(...)" crea un OBJETO (instancia) de la clase.
// Cada objeto tiene sus propias copias de las propiedades.

$curso1 = new Curso(
    titulo:         'PHP para principiantes',
    descripcion:    'Aprende PHP desde cero',
    precio:         150.00,
    duracion_horas: 20
);

// Podés crear muchos objetos distintos con el mismo molde
$curso2 = new Curso('Laravel Avanzado', 'Framework completo', 300.00, 40);

// El operador "->" accede a métodos/propiedades del objeto
echo $curso1->resumen();
// → PHP para principiantes — Bs. 150 — 20h

echo $curso1->precioConDescuento(20);
// → 120.0  (aplicó 20% de descuento sobre 150)

// $curso1 y $curso2 son independientes — cambiar uno no afecta al otro
echo $curso2->getPrecio();
// → 300.0
Bloque 1 · Herencia

Herencia — reutilizar código

Una clase hija hereda todo de la clase padre y puede agregar o modificar comportamiento:

📌 ¿Cuándo usar herencia?

Cuando dos clases comparten características pero una es un tipo especial de la otra. CursoGratuito ES UN Curso, pero con precio siempre en 0.

⚠️ Regla importante

Usá parent::__construct() para llamar al constructor de la clase padre. Sin eso, las propiedades heredadas no se inicializan.

La palabra clave extends

Indica que una clase hereda de otra. class Hijo extends Padre

CursoGratuito.php
<?php

// "extends Curso" = CursoGratuito HEREDA todo de Curso.
// Tiene automáticamente: $titulo, $descripcion, $precio,
// $duracion_horas, getTitulo(), getPrecio(), resumen(), etc.
// No tenemos que volver a escribir esas cosas.
class CursoGratuito extends Curso
{
    // Propiedad NUEVA: no existe en la clase padre
    private string $patrocinador;

    public function __construct(
        string $titulo,
        string $descripcion,
        int    $duracion_horas,
        string $patrocinador    // parámetro nuevo
    ) {
        // parent::__construct() llama al constructor del padre (Curso).
        // OBLIGATORIO si queremos inicializar las propiedades heredadas.
        // Fijate que forzamos precio = 0.00 — esto es el "truco"
        // que hace que todos los cursos gratuitos siempre sean gratis.
        parent::__construct(
            $titulo,
            $descripcion,
            0.00,   // precio = 0 siempre, no lo recibe como parámetro
            $duracion_horas
        );
        $this->patrocinador = $patrocinador;
    }

    // OVERRIDE: reemplazamos el método resumen() del padre.
    // Cuando llamemos $cursoGratuito->resumen() PHP usará
    // ESTE método, no el de Curso.
    // Esto es POLIMORFISMO: mismo método, distinto comportamiento.
    public function resumen(): string
    {
        // getTitulo() lo heredamos de Curso (es public)
        return "{$this->getTitulo()} — GRATIS — Patrocina: {$this->patrocinador}";
    }
}

// ── USO ────────────────────────────────────────────────────
$gratis = new CursoGratuito(
    'Git Básico',
    'Control de versiones',
    5,
    'Universidad'
);
echo $gratis->resumen();
// → Git Básico — GRATIS — Patrocina: Universidad

echo $gratis->getPrecio();
// → 0  (heredado del padre, pero forzado a 0 en el constructor)
Bloque 1 · Contratos

Interface — el contrato

Una interface define qué métodos debe tener una clase, sin decir cómo los implementa. Es un contrato obligatorio.

💡 ¿Por qué usar interfaces?

En el sistema de ventas, diferentes métodos de pago (tarjeta, QR, transferencia) deben tener todos el mismo método procesar(). La interface garantiza eso.

Interface

Define qué hay que hacer. No implementa nada. Se usa con implements.

Abstract

Puede implementar algunos métodos y dejar otros incompletos. Se usa con extends.

MetodoPago.php
<?php

// INTERFACE: es un "contrato" — no tiene código, solo firmas.
// Toda clase que lo implemente DEBE tener estos métodos.
// Si falta alguno, PHP lanza un error fatal al cargar la clase.
interface MetodoPago
{
    // Firma del método — sin cuerpo, solo la declaración
    public function procesar(float $monto): bool;
    public function getNombre(): string;
}

// "implements MetodoPago" = esta clase firma el contrato.
// PHP verifica que tenga TODOS los métodos de la interface.
// PagoTarjeta decide CÓMO procesa (con su lógica propia).
class PagoTarjeta implements MetodoPago
{
    public function procesar(float $monto): bool
    {
        // Acá iría la lógica real: llamar a la API del banco, etc.
        // Por ahora simulamos con un echo
        echo "Procesando Bs. {$monto} con tarjeta...";
        return true; // true = pago exitoso
    }
    public function getNombre(): string
    {
        return 'Tarjeta de débito';
    }
}

// Otra clase que implementa el MISMO contrato pero con distinta lógica.
// Esto es el poder de las interfaces: el código que llama a procesar()
// no sabe si es tarjeta o QR — solo sabe que va a funcionar.
class PagoQR implements MetodoPago
{
    public function procesar(float $monto): bool
    {
        echo "Generando QR por Bs. {$monto}...";
        return true;
    }
    public function getNombre(): string
    {
        return 'Código QR';
    }
}

// Uso polimórfico: la función acepta CUALQUIER MetodoPago
function cobrar(MetodoPago $metodo, float $monto): void
{
    echo "Usando: " . $metodo->getNombre();
    $metodo->procesar($monto);
}
cobrar(new PagoTarjeta(), 150.00);
cobrar(new PagoQR(),      150.00);

<?php

// Laravel organiza el código por "namespaces" (espacios de nombres).
// Esto evita conflictos entre clases con el mismo nombre.
namespace App\Models;

// Importamos la clase base de Eloquent.
// "use" es como un "import" en otros lenguajes.
use Illuminate\Database\Eloquent\Model;

// Nuestro modelo extiende Model de Eloquent.
// Eso nos da GRATIS: all(), find(), create(), save(), delete()...
// Laravel asume que esta clase corresponde a la tabla "cursos"
// (convención: modelo en singular, tabla en plural).
class Curso extends Model
{
    // Aquí definiremos los campos y relaciones
    // en los siguientes pasos
}
⚠️ Nombre en singular — siempre

El modelo se llama Curso (singular). Laravel automáticamente busca la tabla cursos (plural). Si el nombre no sigue esta convención, tenés que definir la tabla manualmente.

Bloque 3 · Paso 3

Migración: tabla usuarios

Abrí el archivo de migración de usuarios en database/migrations/ y reemplazá su contenido con esto:

database/migrations/xxxx_create_usuarios_table.php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

// Las migraciones son clases anónimas en Laravel 9+.
// Definen cómo crear (up) y deshacer (down) cambios en la DB.
return new class extends Migration
{
    // up() se ejecuta cuando hacés "php artisan migrate"
    // Define la estructura de la tabla que querés crear.
    public function up(): void
    {
        // Schema::create(nombre_tabla, función_con_columnas)
        Schema::create('usuarios', function (Blueprint $table) {

            // id() crea una columna "id" de tipo BIGINT UNSIGNED,
            // auto-incremental y clave primaria. Equivale a:
            // `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY
            $table->id();

            // string() = VARCHAR 255 por defecto
            $table->string('nombre');

            // unique() agrega un índice UNIQUE — no pueden repetirse emails
            $table->string('email')->unique();

            $table->string('password'); // se guardará encriptado (bcrypt)

            // enum() restringe los valores posibles a una lista.
            // default() define el valor cuando no se especifica nada.
            $table->enum('rol', ['estudiante', 'instructor', 'admin'])
                   ->default('estudiante');

            // nullable() = la columna acepta NULL (campo opcional)
            $table->string('telefono')->nullable();
            $table->text('bio')->nullable(); // TEXT = texto largo

            // timestamps() crea DOS columnas automáticamente:
            // "created_at" y "updated_at" (ambas TIMESTAMP NULL)
            // Laravel las actualiza solo — no tenés que hacerlo a mano.
            $table->timestamps();
        });
    }

    // down() se ejecuta cuando hacés "php artisan migrate:rollback"
    // Debe DESHACER lo que hizo up() — en este caso, borrar la tabla.
    public function down(): void
    {
        Schema::dropIfExists('usuarios');
    }
};
Bloque 3 · Paso 4

Migración: tabla cursos

database/migrations/xxxx_create_cursos_table.php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('cursos', function (Blueprint $table) {
            $table->id();

            // foreignId() crea la columna de clave foránea.
            // constrained('usuarios') indica que referencia al id de la tabla usuarios.
            // onDelete('cascade') = si se borra el instructor, se borran sus cursos.
            // Equivale a: FOREIGN KEY (instructor_id) REFERENCES usuarios(id) ON DELETE CASCADE
            $table->foreignId('instructor_id')
                   ->constrained('usuarios')
                   ->onDelete('cascade');

            $table->string('titulo');
            $table->text('descripcion');

            // decimal(columna, total_dígitos, decimales)
            // decimal('precio', 8, 2) → hasta 999999.99 (8 dígitos, 2 decimales)
            // default(0) → precio gratuito si no se especifica
            $table->decimal('precio', 8, 2)->default(0);

            $table->integer('duracion_horas'); // número entero de horas
            $table->string('imagen')->nullable(); // ruta a la imagen (opcional)

            // Estado del ciclo de vida del curso.
            // Un instructor crea el curso en 'borrador', lo revisa,
            // y cuando está listo lo pone en 'publicado'.
            $table->enum('estado', ['borrador', 'publicado', 'archivado'])
                   ->default('borrador');

            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('cursos');
    }
};
⚠️ Orden importa

cursos depende de usuarios (por el instructor_id). La migración de usuarios debe ejecutarse primero. Laravel ordena por fecha en el nombre del archivo — por eso el nombre empieza con timestamp.

Migraciones: ventas e inscripciones

xxxx_create_ventas_table.php
public function up(): void
{
    Schema::create('ventas', function(Blueprint $table) {
        $table->id();

        // Quién compró: referencia al usuario/estudiante
        $table->foreignId('usuario_id')
               ->constrained('usuarios')
               ->onDelete('cascade');

        // Qué compró: referencia al curso
        $table->foreignId('curso_id')
               ->constrained('cursos')
               ->onDelete('cascade');

        // Total pagado en el momento de la compra.
        // Guardamos el precio aquí porque el precio del
        // curso podría cambiar después — queremos saber
        // cuánto pagó el estudiante en ese momento.
        $table->decimal('total', 8, 2);

        // Estado del proceso de pago
        $table->enum('estado', [
            'pendiente',  // esperando pago
            'pagado',     // pago confirmado
            'cancelado'   // venta cancelada
        ])->default('pendiente');

        $table->timestamps(); // fecha de la venta en created_at
    });
}
xxxx_create_inscripciones_table.php
public function up(): void
{
    Schema::create('inscripciones', function(Blueprint $table) {
        $table->id();

        // Este es el estudiante inscrito
        $table->foreignId('usuario_id')
               ->constrained('usuarios')
               ->onDelete('cascade');

        // El curso al que está inscrito
        $table->foreignId('curso_id')
               ->constrained('cursos')
               ->onDelete('cascade');

        // Porcentaje de avance (0 a 100).
        // La app actualizará este número a medida
        // que el estudiante complete lecciones.
        $table->integer('progreso')->default(0);

        // true cuando progreso llega a 100%
        $table->boolean('completado')->default(false);

        $table->timestamps();

        // ÍNDICE ÚNICO COMPUESTO:
        // Un estudiante NO puede inscribirse dos veces al mismo curso.
        // Equivale a: UNIQUE KEY (usuario_id, curso_id)
        // Si intentás duplicar, MySQL lanzará un error.
        $table->unique(['usuario_id', 'curso_id']);
    });
}

"Table already exists" — significa que ya ejecutaste la migración antes. Usá php artisan migrate:fresh para borrar todo y volver a crear. Cuidado: borra todos los datos.

⚠️ Otro error frecuente

"Foreign key constraint" — el orden de las migraciones está mal. La tabla referenciada debe existir antes. Revisá las fechas en los nombres de los archivos.

✅ Verificar en phpMyAdmin

Abrí localhost/phpmyadmin → seleccioná cursos_db. Deberías ver todas las tablas listadas en el panel izquierdo.

Definir relaciones en los modelos

Eloquent permite navegar entre tablas usando métodos de relación. Editá cada modelo:

app/Models/Usuario.php
<?php
namespace App\Models;
use Illuminate\Database\Eloquent\Model;

class Usuario extends Model
{
    // Le decimos a Eloquent el nombre exacto de la tabla.
    // Necesario porque "Usuario" pluraliza a "usuarios",
    // pero si hubiera ambigüedad, mejor ser explícito.
    protected $table = 'usuarios';

    // $fillable lista los campos que se pueden llenar
    // con create() o fill(). Es una medida de seguridad:
    // evita que el usuario envíe campos que no queremos
    // (como "rol=admin" en un formulario de registro).
    protected $fillable = [
        'nombre', 'email', 'password', 'rol'
    ];

    // hasMany = "un usuario tiene muchos cursos (como instructor)"
    // Laravel busca la FK "instructor_id" en la tabla cursos.
    // Uso: $usuario->cursos  → colección de cursos del instructor
    public function cursos()
    {
        return $this->hasMany(Curso::class, 'instructor_id');
    }

    // Ventas realizadas por este usuario (como estudiante)
    public function ventas()
    {
        return $this->hasMany(Venta::class, 'usuario_id');
    }

    // belongsToMany = relación muchos-a-muchos a través de "inscripciones".
    // withPivot() nos da acceso a columnas extra de la tabla intermedia.
    public function inscripciones()
    {
        return $this->belongsToMany(
            Curso::class, 'inscripciones'
        )->withPivot('progreso', 'completado');
        // Uso: $usuario->inscripciones[0]->pivot->progreso → 45
    }
}
app/Models/Curso.php
<?php
namespace App\Models;
use Illuminate\Database\Eloquent\Model;

class Curso extends Model
{
    protected $fillable = [
        'titulo', 'descripcion', 'precio',
        'duracion_horas', 'instructor_id', 'estado'
    ];

    // belongsTo = "un curso pertenece a un usuario (instructor)"
    // Es la relación inversa de hasMany.
    // Uso: $curso->instructor  → el objeto Usuario del instructor
    // Uso: $curso->instructor->nombre  → nombre del instructor
    public function instructor()
    {
        return $this->belongsTo(Usuario::class, 'instructor_id');
    }

    // Un curso puede tener muchas ventas (comprado por muchos estudiantes)
    public function ventas()
    {
        return $this->hasMany(Venta::class);
    }

    // Relación inversa de inscripciones en Usuario.
    // Nos permite obtener los estudiantes de un curso.
    // Uso: $curso->estudiantes  → colección de Usuarios inscriptos
    public function estudiantes()
    {
        return $this->belongsToMany(
            Usuario::class, 'inscripciones'
        )->withPivot('progreso', 'completado');
    }
}

Consultas con Eloquent — código real

Así se usan los modelos para consultar y guardar datos. Todo esto va en los controladores:

Ejemplos de Eloquent — copiar al controlador
// ══ CREAR ══════════════════════════════════════════════════════
// create() inserta un nuevo registro y devuelve el objeto creado.
// Los campos deben estar en $fillable del modelo.
$curso = Curso::create([
    'titulo'         => 'PHP Avanzado',
    'descripcion'    => 'Curso completo de PHP',
    'precio'         => 200.00,
    'duracion_horas' => 30,
    'instructor_id'  => 1,
]);
// $curso->id  ahora tiene el ID recién generado por la DB

// ══ LEER ═══════════════════════════════════════════════════════

// Trae TODOS los registros de la tabla cursos
$cursos = Curso::all();

// where() filtra por condición → get() ejecuta la consulta
// Genera: SELECT * FROM cursos WHERE estado = 'publicado'
$cursosPublicados = Curso::where('estado', 'publicado')->get();

// findOrFail: busca por ID primario.
// Si no existe → lanza ModelNotFoundException → responde 404
$curso = Curso::findOrFail(1);

// with() = "eager loading": carga la relación en la misma consulta.
// Sin with() → N+1 problema: hace 1 query por cada curso para el instructor.
// Con with()  → 2 queries totales, sin importar cuántos cursos haya.
$cursos = Curso::with('instructor')->get();

// Acceder a la relación cargada (sin hacer otra query a la DB)
echo $curso->instructor->nombre;

// ══ ACTUALIZAR ═════════════════════════════════════════════════
// update() modifica solo los campos indicados
$curso->update(['precio' => 250.00]);

// ══ ELIMINAR ═══════════════════════════════════════════════════
// delete() elimina el registro de la DB
$curso->delete();

// ══ CONSULTA AVANZADA ══════════════════════════════════════════
// Se pueden encadenar múltiples métodos (query builder)
$cursos = Curso::whereBetween('precio', [100, 300])
                ->orderBy('titulo')        // orden alfabético
                ->paginate(10);            // 10 resultados por página
// En la vista: {{ $cursos->links() }} genera los botones de paginación

// ══ MUCHOS A MUCHOS ════════════════════════════════════════════
// attach() inserta en la tabla intermedia (inscripciones)
// con datos extra en la columna pivot
$usuario->inscripciones()->attach($curso->id, [
    'progreso'   => 0,
    'completado' => false,
]);
// detach($id) elimina la inscripción
// sync([$id1, $id2]) sincroniza — agrega los que faltan, borra los sobrantes

Seeders — poblar la base de datos con datos de prueba

Los seeders insertan datos iniciales para probar la aplicación sin cargarlos manualmente:

terminal — crear seeder
# Crea el archivo en database/seeders/CursoSeeder.php
php artisan make:seeder CursoSeeder
database/seeders/CursoSeeder.php
<?php
namespace Database\Seeders;

use App\Models\Curso;
use App\Models\Usuario;
use Illuminate\Database\Seeder;

class CursoSeeder extends Seeder
{
    // run() es el método que ejecuta artisan db:seed
    public function run(): void
    {
        // Creamos un usuario instructor de prueba.
        // bcrypt() encripta la contraseña — NUNCA guardes
        // contraseñas en texto plano en la base de datos.
        $instructor = Usuario::create([
            'nombre'   => 'Docente Demo',
            'email'    => 'docente@demo.com',
            'password' => bcrypt('password123'),
            'rol'      => 'instructor',
        ]);

        // Array de datos para crear varios cursos en un loop
        $cursos = [
            ['titulo'=>'PHP Básico',    'precio'=>100, 'duracion_horas'=>20],
            ['titulo'=>'Laravel Pro',    'precio'=>250, 'duracion_horas'=>40],
            ['titulo'=>'MySQL Avanzado', 'precio'=>180, 'duracion_horas'=>25],
        ];

        // El operador "..." (spread) expande el array $data
        // como si fueses escribiendo cada par key=>value manualmente
        foreach ($cursos as $data) {
            Curso::create([
                ...$data,
                'descripcion'   => 'Descripción del curso',
                'instructor_id' => $instructor->id,
                'estado'        => 'publicado',
            ]);
        }
    }
}
terminal — ejecutar seeders
# Ejecuta DatabaseSeeder (que puede llamar a otros seeders)
php artisan db:seed

# O ejecutar solo un seeder específico
php artisan db:seed --class=CursoSeeder

# El comando más útil en desarrollo:
# borra todo, recrea las tablas y carga los datos de prueba
php artisan migrate:fresh --seed
