¿Qué son HTML, CSS y JS?

Toda página web que ves en tu navegador está construida con tres tecnologías que trabajan juntas. No son opcionales: son los tres pilares de la web.

Tecnología
Rol
🧱 HTML
La estructura. Es el esqueleto. Define qué hay en la página: títulos, párrafos, imágenes, botones.
🎨 CSS
El diseño. Es la ropa. Define cómo se ve: colores, fuentes, espacios, animaciones.
⚡ JavaScript
La lógica. Es el cerebro. Define qué hace: reaccionar a clics, cargar datos, mostrar/ocultar cosas.

💡 La analogía clásica: HTML es la casa (paredes, puertas, ventanas), CSS es la decoración (pintura, muebles), JS es la electricidad (luces que se encienden, puertas que se abren).

// 01 — estructura

HTML

HyperText Markup Language. No es un lenguaje de programación, es un lenguaje de marcado. Usás etiquetas (tags) para decirle al navegador qué es cada cosa.

Anatomía de una etiqueta

<p class="intro">Hola mundo</p>

<!--  ↑ tag de apertura      ↑ contenido   ↑ tag de cierre  -->
<!--       ↑ atributo = valor                               -->

Algunas etiquetas son auto-cerrantes (no tienen contenido interno):

<img src="foto.jpg" alt="Mi foto" />
<br />
<input type="text" />

Estructura base de toda página HTML

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Mi página</title>
  </head>
  <body>
    <!-- Todo lo visible va aquí -->
  </body>
</html>

📌 <head> = configuración invisible (título, fuentes, meta datos). <body> = todo lo que el usuario ve.

Los tags más importantes

<h1> ... <h6>
Títulos. h1 es el más importante, h6 el menor. Jerarquía del contenido.
<p>
Párrafo de texto. El bloque de texto más común.
<a>
Enlace (anchor). Usa el atributo href para la URL de destino.
<img>
Imagen. Requiere src (ruta) y alt (texto alternativo).
<ul> / <ol>
Lista sin orden / con orden. Sus ítems son <li>.
<div>
Contenedor genérico en bloque. El más usado para agrupar.
<span>
Contenedor genérico en línea. Para estilizar texto parcialmente.
<button>
Botón clickeable. Siempre usalo para acciones.
<input>
Campo de formulario. El atributo type define si es texto, número, checkbox, etc.
<form>
Formulario que agrupa inputs y los envía.
<table>
Tabla de datos. Con <tr> (filas), <th> (cabecera), <td> (celdas).
<strong> / <em>
Texto en negrita / cursiva con significado semántico.

HTML vs HTML5

HTML5 es la versión moderna (desde 2014). La diferencia más importante: HTML5 agregó etiquetas semánticas que describen el significado del contenido, no solo su apariencia.

HTML (antiguo) viejo
HTML5 (actual) moderno
<div id="header">
<header> — semántico, claro
<div id="nav">
<nav> — describe la navegación
<div id="content">
<main> / <article> / <section>
<div id="footer">
<footer> — semántico
Sin video nativo
<video src="..."> nativo
Sin audio nativo
<audio src="..."> nativo

Ejemplo práctico — La página cruda (solo HTML)

Vamos a construir una tarjeta de perfil. Sin CSS se va a ver "trucha" pero va a funcionar. Este es el punto de partida:

▶ Vista previa — Solo HTML (sin estilos)

👆 ¿Ves? La estructura está ahí, toda la información está, pero se ve básica. El navegador aplica estilos por defecto. Ahora entra CSS.

// 02 — diseño

CSS

Cascading Style Sheets. Define cómo se ve cada elemento HTML. La palabra "cascading" (en cascada) refiere a cómo los estilos se aplican en orden de especificidad y herencia.

Sintaxis básica

selector {
  propiedad: valor;
  otra-propiedad: otro-valor;
}

Los tres tipos de selectores

/* Por etiqueta — aplica a TODOS los <p> */
p {
  color: gray;
}

/* Por clase — aplica a elementos con class="destacado" */
.destacado {
  color: blue;
  font-weight: bold;
}

/* Por ID — aplica al elemento con id="perfil" (único) */
#perfil {
  background: white;
  padding: 20px;
}

¿Cómo se incluye en HTML?

<!-- Opción 1: Archivo externo (recomendado) -->
<link rel="stylesheet" href="estilos.css">

<!-- Opción 2: Bloque <style> en el <head> -->
<style>
  body { background: #f0f0f0; }
</style>

<!-- Opción 3: Inline (evitar, difícil de mantener) -->
<p style="color: red;">Texto rojo</p>

Las propiedades más usadas

color
Color del texto. Acepta nombres, hex (#fff), rgb(), hsl().
background
Color o imagen de fondo. También background-color, background-image.
font-size
Tamaño de texto. En px, rem, em, %, vw.
font-family
Fuente tipográfica. Siempre poner una alternativa genérica.
padding
Espacio interno (entre el borde y el contenido).
margin
Espacio externo (entre el elemento y los demás).
border
Borde del elemento: grosor, tipo, color.
border-radius
Esquinas redondeadas. 50% = círculo perfecto.
display
Cómo se muestra el elemento. block, inline, flex, grid.
flexbox
display: flex — sistema de layout unidimensional (fila o columna).
width / height
Ancho y alto del elemento. También max-width, min-height.
transition
Anima cambios de propiedades suavemente.

El modelo de caja (Box Model)

Cada elemento HTML es una caja. Entender esto es fundamental:

.caja {
  width: 200px;        /* ancho del contenido */
  padding: 20px;       /* espacio interno */
  border: 2px solid #333; /* borde */
  margin: 10px;        /* espacio externo */
  /* Total visual = 200 + 20*2 + 2*2 = 244px */
  box-sizing: border-box; /* ← truco: el padding se incluye en el width */
}

// 03 — interactividad

JavaScript

El único lenguaje de programación del navegador. Con JS podés reaccionar a eventos, manipular el HTML y CSS dinámicamente, validar formularios, hacer pedidos a servidores, y mucho más.

¿Cómo se incluye?

<!-- Al final del body (recomendado) -->
<script src="script.js"></script>

<!-- O inline en el HTML -->
<script>
  console.log("Hola mundo");
</script>

Conceptos fundamentales

// Variables
let nombre = "María";       // puede cambiar
const edad = 28;           // no puede cambiar

// Funciones
function saludar(persona) {
  return `Hola, ${persona}!`;
}

// Condicionales
if (edad >= 18) {
  console.log("Es mayor de edad");
} else {
  console.log("Es menor");
}

// Arrays y bucles
const skills = ["HTML", "CSS", "JS"];
skills.forEach(s => console.log(s));

El DOM — Document Object Model

El DOM es la representación del HTML como un árbol de objetos que JS puede modificar. Es el puente entre JS y la página.

// Seleccionar elementos
const boton = document.querySelector("button");
const perfil = document.querySelector("#perfil");
const titulos = document.querySelectorAll("h2");

// Leer / modificar contenido
perfil.textContent = "Nuevo texto";
perfil.innerHTML = "<strong>HTML adentro</strong>";

// Cambiar estilos
boton.style.background = "red";

// Agregar / quitar clases CSS
perfil.classList.add("activo");
perfil.classList.remove("oculto");
perfil.classList.toggle("destacado");

Eventos — Escuchar al usuario

const boton = document.querySelector("#btn");

// El evento "click" se dispara cuando el usuario hace clic
boton.addEventListener("click", function() {
  alert("¡Mensaje enviado!");
});

// Con arrow function (más moderno)
boton.addEventListener("click", () => {
  boton.textContent = "✓ Enviado";
  boton.style.background = "#27ae60";
});

// Otros eventos comunes
input.addEventListener("input", () => { ... });    // al escribir
form.addEventListener("submit", (e) => { ... });   // al enviar
div.addEventListener("mouseover", () => { ... });  // al pasar el mouse

Ejemplo completo — HTML + CSS + JS interactivo

Acá está la tarjeta con funcionalidad real: el botón cambia de estado, se puede editar el nombre y hay un contador de visitas.

// Seleccionamos los elementos
const btn = document.querySelector("#btn");
const nombreEl = document.querySelector("#nombre");
const contador = document.querySelector("#contador");
let clicks = 0;

// Botón de contacto — cambia al hacer clic
btn.addEventListener("click", () => {
  clicks++;
  btn.textContent = `✓ Contactado (${clicks})`;
  btn.style.background = "#27ae60";
  contador.textContent = `Clicks: ${clicks}`;

  // Vuelve al estado original después de 2 segundos
  setTimeout(() => {
    btn.textContent = "Contactar";
    btn.style.background = "#3498db";
  }, 2000);
});

// Hacer el nombre editable con doble clic
nombreEl.addEventListener("dblclick", () => {
  const nuevo = prompt("Nuevo nombre:", nombreEl.textContent);
  if (nuevo) nombreEl.textContent = nuevo;
});
▶ Vista previa — HTML + CSS + JS (¡probalo! hacé clic en el botón)

⚡ ¡Probalo! Hacé clic en el botón, esperá 2 segundos y volvé a hacer clic. También podés hacer doble clic en el nombre para editarlo. Eso es JavaScript en acción.

Resumen del flujo completo

<!DOCTYPE html>
<html>
<head>
  <!-- 1. Conectamos el CSS -->
  <link rel="stylesheet" href="estilos.css">
</head>
<body>

  <!-- 2. Estructura HTML -->
  <button id="btn">Contactar</button>

  <!-- 3. Conectamos el JS (al final del body) -->
  <script src="script.js"></script>
</body>
</html>

¿Y ahora qué?

Ya entendés los tres pilares. La web entera está construida con estos tres lenguajes. El camino sugerido:

Nivel
Qué aprender
🟠 HTML sólido
Formularios, tablas, accesibilidad (ARIA), SEO semántico.
🟢 CSS intermedio
Flexbox en profundidad, CSS Grid, variables CSS, animaciones, responsive design con media queries.
🟡 JS intermedio
Async/await, fetch API, ES6+, manipulación avanzada del DOM, localStorage.
⚡ Frameworks
React, Vue o Svelte para JS. Tailwind CSS para estilos.

💡 El mejor aprendizaje es haciendo. Tomá un proyecto real (una página personal, una lista de tareas, un quiz) y construilo desde cero con HTML + CSS + JS.
