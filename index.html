Contexto

¿Qué es React?

Antes de instalar cualquier cosa, entendamos qué problema resuelve.

📚

¿Qué es?

Una biblioteca de JavaScript creada por Meta (Facebook) en 2013 para construir interfaces de usuario.

🎯

¿Para qué sirve?

Construir interfaces web rápidas e interactivas donde los datos cambian sin recargar la página entera.

🌍

¿Quién lo usa?

Facebook, Instagram, Netflix, Airbnb, Uber, WhatsApp Web. Es la biblioteca front-end más popular del mundo.

La idea central

En lugar de manipular el HTML directamente con document.getElementById(), con React describís cómo debería verse la pantalla según tus datos y React se encarga de actualizar el DOM automáticamente.

Motivación

JavaScript puro vs. React

¿Por qué no usar solo JavaScript? Mirá la diferencia:

❌ JavaScript puro — difícil de mantener
// Tenés que manipular el DOM manualmente
let contador = 0;

function sumar() {
  contador = contador + 1;
  // Tenés que buscar el elemento
  const el = document.getElementById('num');
  // Y actualizarlo manualmente
  el.textContent = contador;
}
✅ React — declarativo y simple
import { useState } from 'react';

function Contador() {
  const [contador, setContador] = useState(0);
  // React actualiza la UI solo
  return <button onClick={() => setContador(contador+1)}>
    {contador}
  </button>;
}
🔑 Concepto clave

Con JavaScript puro, vos le decís "cómo" cambiar el DOM. Con React, le decís "qué" mostrar según los datos. React decide cómo hacerlo eficientemente.

Fase 2 · Concepto

¿Qué es un Componente?

Un componente es simplemente una función de JavaScript que devuelve HTML (JSX). Eso es todo.

  • 1 El nombre siempre empieza con mayúscula: MiComponente, no miComponente
  • 2 Siempre devuelve JSX: una sola cosa (un elemento raíz)
  • 3 Se usa como una etiqueta HTML: <MiComponente />
  • 4 Se puede reutilizar tantas veces como quieras
src/App.jsx
// Esto es un componente
function Saludo() {
  return (
    <div>
      <h1>¡Hola Mundo!</h1>
      <p>Mi primer componente React</p>
    </div>
  );
}

// App usa el componente Saludo
export default function App() {
  return (
    <div>
      <Saludo />
      <Saludo />  ← reutilizado!
      <Saludo />
    </div>
  );
}
Fase 2 · JSX

Reglas de JSX

JSX parece HTML pero tiene diferencias importantes. Estas reglas son las que más confunden al principio:

❌ Incorrecto
// Dos elementos raíz — ERROR
return (
  <h1>Título</h1>
  <p>Párrafo</p>
);

// class → ERROR en JSX
<div class="caja">...</div>

// Etiqueta no cerrada → ERROR
<img src="foto.jpg">
✅ Correcto
// Un solo elemento raíz — OK
return (
  <>
    <h1>Título</h1>
    <p>Párrafo</p>
  </>
);

// className en JSX (no class)
<div className="caja">...</div>

// Siempre cerrar las etiquetas
<img src="foto.jpg" />
💡 JavaScript dentro de JSX

Para incluir expresiones de JavaScript dentro del HTML, usás llaves { }. Por ejemplo: <h1>{nombre}</h1> o <p>{2 + 2}</p> mostraría "4".

Fase 2 · Props

Props: pasando datos entre componentes

Las props son parámetros que el componente padre le pasa al hijo. Funcionan igual que los atributos HTML.

📌 Concepto

Las props van de padre a hijo, nunca al revés. Son de solo lectura — el hijo no puede modificarlas directamente.

Analogía

Pensá en un componente como una función:
Tarjeta(nombre, edad)

Las props son los argumentos que le pasás a esa función para personalizar su comportamiento.

src/App.jsx — Copiá y pegá esto
// Componente que recibe props
function Tarjeta({ nombre, carrera, año }) {
  return (
    <div style={{
      border: '1px solid #61dafb',
      padding: '1rem',
      borderRadius: '8px',
      margin: '8px'
    }}>
      <h2>{nombre}</h2>
      <p>Carrera: {carrera}</p>
      <p>Año: {año}</p>
    </div>
  );
}

// App pasa props a cada Tarjeta
export default function App() {
  return (
    <>
      <Tarjeta
        nombre="Ana García"
        carrera="Sistemas"
        año={3}
      />
      <Tarjeta
        nombre="Luis Mamani"
        carrera="Informática"
        año={1}
      />
    </>
  );
}
Fase 2 · Listas

Renderizar listas con .map()

Muy frecuente: tenés un array de datos y querés mostrar un componente por cada elemento.

⚠️ La prop key — obligatoria

Cada elemento de una lista necesita una key única. React la usa internamente para saber qué cambió. Sin ella, el código funciona pero React muestra advertencias.

💡 ¿Por qué .map() y no un for?

Dentro de JSX no podés usar sentencias (for, if) — solo expresiones. .map() es una expresión que devuelve un array de JSX.

src/App.jsx
export default function App() {
  // Un array de datos
  const estudiantes = [
    { id: 1, nombre: 'Ana',  nota: 85 },
    { id: 2, nombre: 'Luis', nota: 92 },
    { id: 3, nombre: 'María',nota: 78 },
  ];

  return (
    <ul>
      {estudiantes.map(estudiante => (
        <li key={estudiante.id}>
          {estudiante.nombre} — Nota: {estudiante.nota}
        </li>
      ))}
    </ul>
  );
}

Fase 3 · useState

El hook useState

Un hook es una función especial de React. useState es el más importante — permite que un componente tenga su propio estado.

📌 La anatomía de useState
const [valor, setValor] = useState(inicial);
   ↑            ↑                   ↑
 estado    función para         valor
 actual     cambiarlo           inicial
⚠️ Regla crítica

Nunca modifiques el estado directamente. Esto no funciona: valor = 5. Siempre usá la función setter: setValor(5). Solo así React sabe que debe re-renderizar.

src/App.jsx — Contador interactivo
import { useState } from 'react';

export default function App() {
  // Estado: contador empieza en 0
  const [contador, setContador] = useState(0);

  return (
    <div style={{ textAlign: 'center', padding: '2rem' }}>
      <h1>Contador: {contador}</h1>

      <button onClick={() => setContador(contador + 1)}>
        + Sumar
      </button>

      <button onClick={() => setContador(contador - 1)}>
        - Restar
      </button>

      <button onClick={() => setContador(0)}>
        Reset
      </button>
    </div>
  );
}
Fase 3 · Cómo funciona

¿Qué pasa cuando cambia el estado?

Es importante entender el ciclo que ocurre cada vez que el usuario hace algo:

👆

1. Evento

El usuario hace clic en un botón

⚡

2. setState()

Se llama la función setter con el nuevo valor

🔄

3. Re-render

React vuelve a ejecutar la función del componente

🖥

4. UI actualizada

El DOM se actualiza solo con los cambios mínimos

💡 Virtual DOM — por qué React es eficiente

React no actualiza todo el HTML cada vez. Mantiene una copia virtual del DOM en memoria, compara qué cambió (diffing) y solo actualiza las partes que cambiaron. Esto lo hace muy rápido incluso con datos complejos.

Fase 3 · Formularios

Formularios e inputs controlados

En React, los inputs del formulario deben estar sincronizados con el estado. A esto se le llama componente controlado.

📌 ¿Qué es un input controlado?

El value del input viene del estado de React. El estado es la "fuente de verdad". Cada vez que el usuario escribe, onChange actualiza el estado, que actualiza el input.

Flujo del input controlado

Usuario escribe "A"
   ↓
onChange dispara: setTexto("A")
   ↓
Estado: texto = "A"
   ↓
Input muestra: value="A"
src/App.jsx
import { useState } from 'react';

export default function App() {
  const [nombre, setNombre] = useState('');

  return (
    <div>
      // El input está CONTROLADO por el estado
      <input
        value={nombre}
        onChange={e => setNombre(e.target.value)}
        placeholder="Escribí tu nombre"
      />

      // Se actualiza EN TIEMPO REAL
      <p>
        {nombre ? `Hola, ${nombre}!` : 'Esperando...'}
      </p>
    </div>
  );
}

Fase 4 · Código base

Lista de Tareas — Código completo

Reemplazá todo el contenido de src/App.jsx con este código. Leé cada comentario:

src/App.jsx — Lista de Tareas completa
import { useState } from 'react';

export default function App() {
  // Estado 1: la lista de tareas (array vacío al inicio)
  const [tareas, setTareas] = useState([]);

  // Estado 2: el texto del input
  const [nueva, setNueva] = useState('');

  // Función para agregar una tarea
  const agregarTarea = () => {
    if (!nueva.trim()) return; // no agregar si está vacío
    setTareas([...tareas, { id: Date.now(), texto: nueva }]);
    setNueva(''); // limpiar el input
  };

  // Función para eliminar una tarea por id
  const eliminar = (id) => {
    setTareas(tareas.filter(t => t.id !== id));
  };

  return (
    <div style={{ maxWidth: '400px', margin: '2rem auto' }}>
      <h1>📝 Mis Tareas</h1>

      // Input + botón para agregar
      <div>
        <input
          value={nueva}
          onChange={e => setNueva(e.target.value)}
          placeholder="Nueva tarea..."
          onKeyDown={e => e.key === 'Enter' && agregarTarea()}
        />
        <button onClick={agregarTarea}>Agregar</button>
      </div>

      // Lista de tareas
      <ul>
        {tareas.map(tarea => (
          <li key={tarea.id}>
            {tarea.texto}
            <button onClick={() => eliminar(tarea.id)}>
              Eliminar
            </button>
          </li>
        ))}
      </ul>

      // Contador de tareas
      <p>Total: {tareas.length} tareas</p>
    </div>
  );
}
Fase 4 · Desafíos
3

Desafíos para completar

Extendé la app que creaste. Si terminás uno, pasá al siguiente.

  • A Marcar como completada: Agregá un completada: false al objeto de tarea. Al hacer clic en ella, cambiá ese valor. Si está completada, mostrala con texto tachado (text-decoration: 'line-through').
  • B Filtrar tareas: Agregá botones "Todas", "Pendientes", "Completadas". Guardá el filtro activo en un estado y usá .filter() para mostrar solo las correspondientes.
  • C Persistir datos: Investigá el hook useEffect. Úsalo para guardar las tareas en localStorage cada vez que cambien, y cargarlas al iniciar la app.
🎯 Objetivo del laboratorio

No importa si no terminás todos los desafíos hoy. Lo importante es entender el flujo: estado cambia → React re-renderiza → UI se actualiza. Ese es el corazón de React.
